#  Manual Testing

This section contains examples of **manual testing artifacts** created as part of my QA practice.

The goal of this module is to demonstrate:
- understanding of **manual QA fundamentals**
- ability to design **test cases**
- ability to identify, analyze, and document **software defects**
- basic **QA thinking** at Trainee / Junior level

---

##  Scope of Testing

The tested functionality is **Login feature** of a web application.

During testing, the following aspects were covered:
- positive and negative login scenarios
- field validation
- error handling
- UI / UX behavior
- basic localization checks

---

##  Contents

###  Test Cases
File: `test_cases.md`

Includes:
- 2 positive test cases
- 3 negative test cases
- validation of required fields
- alternative user behavior (Enter key login)

All test cases are written in **Markdown** and structured to be easily readable in GitHub Preview mode.

---

###  Bug Reports
File: `login_bug_reports.md`

Includes:
- 5 bug reports found during manual testing
- functional, validation, UI, UX, and localization defects
- bugs with different **Severity** and **Priority** levels

Each bug report contains:
- Preconditions
- Steps to Reproduce
- Actual Result
- Expected Result
